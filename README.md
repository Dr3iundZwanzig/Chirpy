# Chirpy

Chirpy is a guided Boot.dev project: a lightweight Go microblogging API for creating, reading, and deleting short messages called "chirps." It includes user registration, JWT authentication, refresh token support, and webhook-based upgrades for premium users.

## Features

- User signup with password hashing
- JWT-based login and refresh token rotation
- Create, list, retrieve, and delete chirps
- Content filtering for blocked words in chirp bodies
- Premium user upgrade via authenticated webhook events
- Health check and admin metrics page
- Development reset endpoint for local workflows

## Architecture

- `main.go` sets up HTTP routing and server configuration
- `internal/auth` handles password hashing, JWT creation/validation, and token parsing
- `internal/database` contains SQL model bindings generated by `sqlc`
- `handler*.go` files implement request handling for users, chirps, auth, refresh tokens, webhooks, and admin utilities

## Getting Started

### Requirements

- Go 1.25 or later
  - Install with your package manager, for example on Ubuntu:
    ```bash
    sudo apt update
    sudo apt install golang-go
    ```
- PostgreSQL database
  - Install with your package manager, for example on Ubuntu:
    ```bash
    sudo apt update
    sudo apt install postgresql postgresql-contrib
    ```
- `sqlc` (optional, only if you regenerate database bindings)
  - Install with `go install`:
    ```bash
    go install github.com/kyleconroy/sqlc/cmd/sqlc@latest
    ```

### Environment

Create a `.env` file in the project root with the following values:

```env
SECRET=your_jwt_secret
POLKA_KEY=your_webhook_api_key
DB_URL=postgres://user:password@localhost:5432/chirpy?sslmode=disable
PLATFORM=dev
```

- `SECRET` - secret key used to sign JWTs
- `POLKA_KEY` - API key accepted by `/api/polka/webhooks`
- `DB_URL` - PostgreSQL connection string. This value may change depending on your local PostgreSQL setup, username, database name, port, or SSL settings.
- `PLATFORM` - set to `dev` to enable `/admin/reset`

### Install

If you want to install the CLI binary locally, run:

```bash
go install github.com/Dr3iundZwanzig/Chirpy@latest
```

The binary will be installed in your Go bin directory (`$GOBIN` or `$HOME/go/bin`) and can be run as `Chirpy`.

### Run Locally

```bash
go run .
```

The server listens on `http://localhost:8080`.

## Database

This project uses PostgreSQL and SQL bindings generated into `internal/database`.

Tables include:

- `users`
- `chirps`
- `refresh_tokens`

If you modify SQL definitions, regenerate bindings with `sqlc generate`.

## Notes

- Passwords are hashed with Argon2id
- JWT access tokens expire after 1 hour
- Refresh tokens expire after 60 days
- The `/admin/reset` endpoint is intentionally restricted to the `dev` platform only
